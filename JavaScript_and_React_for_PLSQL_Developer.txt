// functions

In JavaScript, functions are "first-class values" which simply means that 
functions are treated just like any other piece of data, such as strings, numbers, 
or arrays.
There is no special VIP treatment for numbers or text; anything you can do to a regular variable, you can also do to a function.
Here is a breakdown of what that looks like in practice, using simple analogies:

1. You can store a function in a variable just like you can assign the number 2026 or 
the string "Hello" to a variable, you can assign a whole function to a variable.

    // Storing a number
    const currentYear = 2026;

    // Storing a function
    const greet = function() {
    return "Hello!";
    };

2. You can pass a function into another function (as an argument).
Think of a function like a package. You can hand it over to another function to use later. 

    // A function that takes another function as an ingredient

    function runTwice(action) {
    action();
    action();
    }

    ... runTwice is a function that accepts another function as parameter, 
    ... then calls that other funtion twice.

    const sayHi = () => console.log("Hi!");

    // We are passing the sayHi function into runTwice
    runTwice(sayHi);

-- note on line 109, 'action' is like a name for a parameter in a pl/sql function 
-- except unlike pl/sql that parameter ('action') can be another function.

3. You can return a function from another function. Just like a machine can build 
and spit out a toy, a function in JavaScript can build and spit out a new function.

    function multiplier(factor) {
    // It returns a brand new function customised by the "factor"
    return function(number) {
        return number * factor;
    };
    }

    const double = multiplier(2); 
    console.log(double(5)); // Output: 10

-- so in this example function 1 "multiplier" 
-- returns function 2 "{return number * factor;}" which gets assigned to "double"

// more on functions

pl/sql syntax...

  FUNCTION GetTax(p_salary NUMBER)
  RETURN NUMBER
  IS
  BEGIN
      RETURN p_salary * 0.2;
  END;

javascript equivalent...

  function getTax(salary) {
      return salary * 0.2;
  }

called like this...

  let tax = getTax(50000);

-- here 'tax' variable - of type number - would be the value returned, not another function.

In PL/SQL you don’t normally think of a function as a variable.
In JavaScript you can literally store a function inside a variable.

  const getTax = function(salary) {
      return salary * 0.2;
  };

/* in this example 'getTax' is a variable containing a function.
   called like this...
*/

let someValue = getTax(50000);

-- 'someValue' again is a numer type variable, but getTax variable is 'executed' because there are parentheses after the name of function.
-- Without parentheses...
    let test = gettax; 
-- ...simply copies the function into a new variable without running it.

// something that may trip you

There’s one arrow-function case that often surprises beginners
returning an object literal.

This does not do what you might expect:

  const createPerson = () => { name: "James" };

You may think that { name: "James" } is an object. 
JavaScript treats the braces as the function body, not as an object, 
so the function returns undefined.

to actually assign the object, wrap it in parentheses

  const createPerson = () => ({ name: "James" });

You may ask, why bother with this syntax when it's easier to do...

  let anotherPerson = { name: "Karen" } 

Both create a object with same structure.

() => ({ name: "James" });  isn't meant for creating a single, static object. 
Its real power comes when you need a factory function—a reusable tool to 
dynamically generate objects, especially when working with array methods in React.

  const userNames = ["James", "Karen", "Alex"];

  // Transforming strings into objects on the fly
  const userObjects = userNames.map(name => ({ id: Date.now(), name: name }));

  'userObjects' is an array of objects, mapped from the array of strings 'userNames'

// callback functions

Consider...

  const JamesNephewsNames = ["Alex", "Theo", "John"];

  JamesNephewsNames.forEach(nephew => {
      console.log(`Hello ${nephew} from your uncle James`);
  });

The callback function is the argument passed to forEach:
   nephew => {...}
It is an arrow function that receives each item from the array and runs for each element.
Or you caould say "A function that you pass to another function, so that the other function can call it later."
Notice something interesting. You never created nephew. map() creates it for you.

When we write...

   nephew => {console.log(nephew);}
  
...you’re not assigning a value to nephew. You’re declaring a parameter.

It’s the same as writing

  function greet(nephew) {console.log(nephew);}

Some other good callback functions (filter, find)

  const longNames = JamesNephewsNames.filter(nephew => {
    return nephew.length > 4;
  });

  const result = JamesNephewsNames.find(nephew => nephew === "Theo");
