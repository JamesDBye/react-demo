// functions

In JavaScript, functions are "first-class values" which simply means that 
functions are treated just like any other piece of data, such as strings, numbers, 
or arrays.
There is no special VIP treatment for numbers or text; anything you can do to a regular variable, you can also do to a function.
Here is a breakdown of what that looks like in practice, using simple analogies:

1. You can store a function in a variable just like you can assign the number 2026 or 
the string "Hello" to a variable, you can assign a whole function to a variable.

    // Storing a number
    const currentYear = 2026;

    // Storing a function
    const greet = function() {
    return "Hello!";
    };

2. You can pass a function into another function (as an argument)
Think of a function like a package. You can hand it over to another function to use later. 
See useEffect above, or

    // A function that takes another function as an ingredient

    function runTwice(action) {
    action();
    action();
    }

    const sayHi = () => console.log("Hi!");

    // We are passing the sayHi function into runTwice
    runTwice(sayHi);

-- note on line 109, 'action' is like a name for a parameter in a pl/sql function 
-- except unlike pl/sql that parameter ('action') can be another funtion.

3. You can return a function from another function. Just like a machine can build 
and spit out a toy, a function in JavaScript can build and spit out a new function.

    function multiplier(factor) {
    // It returns a brand new function customized by the "factor"
    return function(number) {
        return number * factor;
    };
    }

    const double = multiplier(2); 
    console.log(double(5)); // Output: 10

-- so in this example function 1 "multiplier" 
-- returns function 2 "{return number * factor;}" which gets assigned to "double"
