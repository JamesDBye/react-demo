Part 1: JavaScript Fundamentals

Chapter 1: Variables

A variable is simply a named piece of memory that stores a value.

Think of it as a labelled box:

Variable Name │ ▼ ┌───────────┐ │ Value │ └───────────┘

Unlike PL/SQL, JavaScript variables do not have a fixed data type. The same kind of variable can hold a number, a string, an array, an object or even a function.

Examples

Simple variable:

const age = 55;
console.log(age);
const firstName = "James";
console.log(`Hello ${firstName}`);

Array:

const nephews = ["Alex", "Theo", "John"];
nephews │ ▼ ┌───────┬───────┬───────┐ │ Alex │ Theo │ John │ └───────┴───────┴───────┘ 0 1 2
// retrieve a value
console.log(nephews[1]);

Object:

const project = {
    projectCode: "Windmill",
    startDate: "2014-01-01",
    endDate: "2024-12-31"
};
project │ ▼ projectCode → Windmill startDate → 2014-01-01 endDate → 2024-12-31
// retrieve a property
console.log(project.projectCode);

PL/SQL Equivalent

A similar concept in PL/SQL is a RECORD:

TYPE project_rec IS RECORD (
    project_code VARCHAR2(20),
    start_date   DATE,
    end_date     DATE
);

Chapter 2: Functions

PL/SQL Comparison

PL/SQL

FUNCTION GetTax(p_salary NUMBER)
RETURN NUMBER
IS
BEGIN
    RETURN p_salary * 0.2;
END;

JavaScript

function getTax(salary) {
    return salary * 0.2;
}

Chapter 3: Functions as Values

A function can also be stored inside a variable:

const getTax = function(salary) {
    return salary * 0.2;
};

Key Distinction:

  • Calling getTax refers to the function
  • Calling getTax() executes it

First-Class Functions

Functions are "first-class values". You can:

  1. Store them in variables
  2. Pass them into other functions
  3. Return them from other functions

Example

function runTwice(action) {
    action();
    action();
}

const sayHi = () => console.log("Hi!");

runTwice(sayHi);
Common Beginner Question: Where did action come from?

Answer: It is simply a parameter. Unlike PL/SQL, JavaScript parameters can themselves be functions.

Chapter 4: Arrow Functions

Traditional vs Arrow Syntax

Traditional

function add(a,b){
    return a+b;
}

Arrow

const add = (a,b) => {
    return a+b;
};

Implicit Return

const square = x => x * x;

Equivalent to:

const square = x => {
    return x * x;
};

Rule

  • One expression → implicit return
  • Braces {} → write return if you want a value returned
Arrow function │ ▼ Using braces? / \ No Yes │ │ Implicit Write return return yourself

Braces {} mean "this is a block of statements."

Common Mistake #1

const square = x => {
    x * x;
};
Problem: This returns undefined. You cannot omit the RETURN keyword if {} braces are used. 
const square = x => x * x;

This is fine because there are no braces, so return is not required.

Common Mistake #2

const createPerson = () => { name: "James" };
Problem: This returns undefined.

Correct:

const createPerson = () => ({ name: "James" });

Chapter 5: Callbacks

Definition

A callback is a function passed to another function so that the other function can call it later.

Example

const names = ["Alex","Theo","John"];

names.forEach(nephew => {
    console.log(`Hello ${nephew}`);
});
Question: Where did nephew come from?

Answer: You declared it as the callback parameter.

Internally, forEach() effectively does:

callback("Alex");
callback("Theo");
callback("John");

Chapter 6: Arrays

const names = [
    "Alex",
    "Theo",
    "John"
];
  • Arrays contain ordered values
  • Index starts at zero
  • names[0] returns "Alex"

Array Methods

forEach()

Use when you want to perform an action.

map()

Use when you want to transform one array into another.

const greetings = names.map(name => `Hello ${name}`);

filter()

Returns items matching a condition.

const longNames = names.filter(name => name.length > 4);

find()

Returns the first matching item.

const theo = names.find(name => name === "Theo");

Chapter 7: Async/Await

async function loadProjects() {
    const data = await getProjects();
}

The purpose is to write asynchronous code that reads almost like synchronous code. That is to say, "Write code that does multiple things at once in the background, but structure it so it's as easy to read and follow as a simple, step-by-step checklist."

Example: Imagine if a browser froze whilst a large file was being downloaded (synchronous). Instead, modern browsers allow users to initiate the download of a large file (asynchronously) and allow the user to do other things at the same time—such as clicking on a hyperlink and navigating elsewhere.

In React

const data = await getProjects();

setProjects(data);
Important: await can only be used inside an async function.

Chapter 10: Import / Export

export

Makes a function or variable available to other JavaScript files.

export function getProjects() {}

Other files may import and use getProjects().

Chapter 11: Conditional (Ternary) Operator

Syntax

condition ? valueIfTrue : valueIfFalse

Example

task.completed ? "✓" : "○"

This means: if task.completed is true, display ✓; otherwise, display ○

Chapter 5: useEffect

Basic Usage

useEffect(() => {
    fetch(...)
        .then(...)
}, []);

Think: Run this code once when the component first appears.

With Dependencies

useEffect(() => {
    // code to execute
}, [selectedProject])

Database Trigger Analogy

CREATE OR REPLACE TRIGGER trg_project_change
AFTER UPDATE OF project_code ON some_table
FOR EACH ROW
BEGIN
    LoadTasks;
END;

This is conceptually similar to a database trigger.

Conceptual Comparison

  • Database trigger: Executes when data changes in Oracle
  • useEffect: Executes when a piece of React state or a prop changes

Key Differences

  • Oracle trigger → watches database data
  • useEffect → watches React component state/props
  • useState is a bit like a package variable that React manages
  • setState (e.g., setSelectedProject) is the only supported way to change it
  • useEffect is React's mechanism for saying, "Now that this state has changed, execute this additional logic"

Technical Details

Technically, useEffect doesn't watch for changes in the way a trigger continuously watches a table. React re-runs the component after state changes, compares the dependency array with its previous values, and if a dependency has changed, it runs the effect.

Multiple useEffects

A component can have more than one useEffect. This is encouraged when each effect has a different responsibility. For example, from the IPO prototype:

  • Effect 1: Load projects when the page starts
  • Effect 2: Load tasks when the selected project changes

Chapter 6: Optional Chaining (?.)

Purpose

Safely access a property when an object might be null or undefined.

Example

selectedProject?.projectCode

Which means: if selectedProject exists, return its projectCode; otherwise return undefined.

Chapter 7: State Ownership

  • A component should own the state that it is responsible for
  • Child components receive state via props
  • Child components notify parents of changes via callback functions (such as onProjectSelected)
